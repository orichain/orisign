# Contributing

Thank you for your interest in contributing!

## Scope
This project is a research prototype and educational implementation of
SQISIGN-like constructions. Contributions improving clarity, correctness,
documentation, and experimental evaluation are welcome.

## Guidelines
- Keep changes minimal and focused.
- Preserve cryptographic correctness and notation consistency.
- Add comments for non-obvious mathematical steps.
- Avoid introducing production-security claims.

## Workflow
1. Fork the repository.
2. Create a feature branch.
3. Commit with clear messages.
4. Open a pull request describing the change and motivation.

## Code Style
Follow existing formatting and naming conventions.

## Review
All contributions are subject to review and may be rejected if they reduce
clarity, correctness, or research integrity.
